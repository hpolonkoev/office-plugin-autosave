Features

Everything an IT team needs,
nothing users have to think about.

AutoSave for Office is designed for enterprise environments where data integrity matters. It works silently in the background while staying fully configurable by administrators.

Works across all Office apps

A single add-in manifest covers Word, Excel, and PowerPoint. Each host uses the appropriate native save API automatically.

Word Excel PowerPoint

Configurable save interval

IT sets the default interval, minimum and maximum bounds. Users can personalise within those bounds — or you can lock it entirely.

Multi-language support

Ships with English, French, and Dutch. Drop any additional locale JSON file on the server — no rebuild, no redeployment needed.

Zero infrastructure

Fully client-side. No Azure Functions, no database, no backend. Host the static files on any HTTPS server — including GitHub Pages.

Protected View aware

Detects read-only and Protected View documents. Disables the timer silently and shows a localized notice — no crashes, no error dialogs.

Shared runtime background saving

Uses Office Shared Runtime so the autosave timer keeps running even when the task pane is closed — transparently, in the background.

How it works

Up and running in four steps.

Deploy once via Microsoft 365 Admin Center and every user in your tenant gets automatic document saving — with no manual installation.

1

Download the manifest

Grab manifest.xml and update the production URL to your GitHub Pages or static hosting address.

2

Deploy to Microsoft 365

Upload the manifest in the Microsoft 365 Admin Center under Integrated Apps. Target groups or the full tenant.

3

Documents save automatically

The add-in activates on every document open. The timer starts immediately and saves silently in the background.

4

Configure without rebuilding

Edit config/autosave-config.json or add a locale file on the server. Changes take effect on the next document open.

IT Configuration

One JSON file. Full control.

No code changes, no rebuilds. Edit config/autosave-config.json on your server and the new settings take effect the next time a user opens a document. 

{
  "defaultIntervalMinutes": 5,  // default save interval
  "minIntervalMinutes":     2,  // user cannot go below this
  "maxIntervalMinutes":    60,  // user cannot go above this
  "allowUserOverride":    true   // set false to lock interval
}
FieldWhat it controls
defaultIntervalMinutes How often to save (in minutes) for new documents or when no user preference exists.
minIntervalMinutes The lowest interval a user can select. Prevents aggressive saving on shared drives.
maxIntervalMinutes The highest interval a user can select. Ensures a reasonable maximum.
allowUserOverride Set to false to hide the settings panel and lock the interval for all users.
Localization

Speaks your users' language.

The add-in auto-detects the Office display language and loads the matching locale file. Fallback chain: exact locale → base language → English. Zero hardcoded strings.

Ready to deploy?

Follow the four steps below and your entire organization will have automatic document saving within minutes.

Step 01

Download the manifest

Get manifest.xml and update the URL to your hosted address.

Step 02

Host the static files

Push the dist/ folder to GitHub Pages, Azure Static Web Apps, or any HTTPS server.

Step 03

Upload to Admin Center

Go to Microsoft 365 Admin Center → Integrated Apps → Upload custom app → upload the manifest.

Step 04

Target users or tenant

Select specific groups or deploy to the whole tenant. Users get the add-in on their next Office launch.
